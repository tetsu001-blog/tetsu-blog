正直に言うと、私は今まで OpenAI 系のAIをあまり積極的に使ってきませんでした。理由は性能ではなく、AIの軍事利用に対する姿勢です。

今回は、なぜ私が Codex を試すことにしたのか。そして、実際に触ってみて面白いと思った **Codex ペット**について書いていきます。

## Codexを使用するようになった経緯
![](/images/Pasted-image-20260506101023.jpg)

私はこれまで、軍事利用に反対する姿勢を示してきた Anthropic を支持していました。これは今も変わりません。むしろ、Claude には今後も課金して使い続けるつもりです。

## Codex ペットとは

Codex を使っていると、実装、調査、修正、レビュー待ちなど、ひとつの作業が少し長く走ることがあります。そんなときに便利なのが **Codex ペット**です。

Codex ペットは、Codex アプリ上で表示できる小さなフローティングキャラクターです。作業中のスレッドの状態をデスクトップ上で見せてくれるので、Codex のウィンドウを何度も開き直さなくても、いま何が起きているのかをゆるく把握できます。

公式ドキュメントでは、アクティブなスレッド、実行中の作業、入力待ち、レビュー待ちなどの状態や、短い progress prompt を表示できるものとして紹介されています。

使い方はシンプルです。

- 入力欄で `/pet` を実行する
- `Ctrl+K` / `Cmd+K` のコマンドメニューから pet を表示する
- `Settings > Appearance > Pets` で使う pet を選ぶ

表示した pet をクリックすると Codex が開くため、作業状況の確認から会話への復帰までが自然につながります。

こういう小さい機能、私は結構好きです。

## 何がうれしいのか

Codex ペットのよさは、機能そのものが大きいというより、作業体験が少し軽くなるところにあります。

エージェントに作業を任せている間、ユーザーは待つだけではありません。別の資料を見たり、別タスクに移ったり、ブラウザで確認したりします。その間も、pet がデスクトップ上にいることで「まだ動いている」「入力待ちになった」「レビューできる状態になった」といった変化に気づきやすくなります。

通知ほど強くなく、ログほど細かすぎない。ちょうどよい存在感で、Codex の作業を横目で見られるのが魅力です。

AIエージェントって、便利なんですが、処理を任せている間は「今なにしてるんだろう？」となりがちです。そこに小さなキャラクターがいるだけで、待ち時間のストレスが少し減ります。

## オリジナル pet も作れる

Codex ペットは、標準の pet を使うだけでなく、自分用のカスタム pet も作れます。

カスタム pet は、スプライトシートと `pet.json` を用意して Codex の pet フォルダに配置する仕組みです。公式には `hatch-pet` というスキルを使う方法が紹介されており、画像やキャラクター案をもとに、Codex 用のアニメーション付き pet を作れます。

たとえば、キャラクター画像を元にして次のような状態を作れます。

- idle: 待機中の呼吸や瞬き
- running-right / running-left: 左右の移動
- waving: 挨拶
- jumping: ジャンプ
- failed: 失敗時のしょんぼりした反応
- waiting: 入力待ち
- review: レビュー待ち

実際に作るときは、元画像をそのまま縮小するのではなく、小さな pet として読めるように単純化するのが重要です。細かすぎる装飾、淡い影、背景、文字、浮いたエフェクトは、192x208 程度の小さな表示では崩れやすくなります。

## 作ってみた感想

今回は、私の最推しキャラである **アークナイツのテキサス** の SD 画像を参考にしつつ、Codex 用の pet を作りました。

![アークナイツのテキサスのゲーム画面スクリーンショット](/images/arknights-texas-screenshot.png)

参考にしたのは、アークナイツの [テキサス（血掟テキサス）](https://arknights.wikiru.jp/?%E8%A1%80%E6%8E%9F%E3%83%86%E3%82%AD%E3%82%B5%E3%82%B9) です。具体的には、SD 画像を Codex に貼り付けて、そこから小さな pet として見えるように生成していきました。

![](/images/Pasted-image-20260506095743.png)

※画像はゲーム画面画像です。
※この pet は個人制作の非公式ファンメイドです。Yostar / Hypergryph とは関係ありません。また、公式素材そのものを配布する意図はありませんが、SD 画像を参考画像として使っているため、完全なオリジナル作品とは言えません。権利面を考えると、pet データの配布や販売は避け、自分の作業環境で楽しむ範囲に留めるのが安全だと考えています。

生成では、まず基準になる base sprite を作り、その後に状態ごとの横長スプライト列を作成しました。最後にそれらを 8x9 の atlas にまとめ、`spritesheet.webp` と `pet.json` としてパッケージ化します。

作成後は Codex の custom pets に配置し、設定画面から選べるようになります。

自分の作業環境に合わせた pet が出てくるだけで、Codex に作業を任せる時間が少し楽しくなります。こういう「作業環境を自分好みにできる余白」は、地味ですが長く使う上ではかなり大事だと思います。

## できること、できないこと

現時点でカスタム pet 側で主にできるのは、見た目とアニメーションの差し替えです。

一方で、pet をクリックしたときの動作を変えたり、pet 側から任意の吹き出しを自由に出したりする公式のカスタム API は、公開ドキュメント上では確認できません。クリックで Codex が開く挙動は、pet 個別の設定というより Codex アプリ側の標準挙動と考えるのがよさそうです。

つまり、カスタム pet は「独自 UI を作る拡張機能」というより、「Codex の状態表示に自分らしいキャラクターを載せる仕組み」と捉えると扱いやすいです。

## まとめ

今回、OpenAI の Codex を使い始めた理由は、手放しで OpenAI を支持するようになったからではありません。

今でも、AIの軍事利用に対しては強い抵抗感がありますし、その点で Anthropic を支持する気持ちは変わっていません。

ただ、AIツールを使う側としては、ひとつの会社だけに依存する怖さもあります。だからこそ、Codex も実際に触って、自分の目で判断することにしました。

その中で見つけた Codex ペットは、エージェント作業の状態をデスクトップ上でやわらかく見せてくれる面白い機能でした。標準の pet だけでも便利ですが、自分のキャラクターやプロジェクトに合わせた custom pet を作ると、作業環境への愛着がかなり増します。

Codex を日常的に使う人ほど、pet の価値はじわじわ効いてくると思います。作業中のエージェントをただのログではなく、少し表情のある存在として見られる。そこが Codex ペットの面白いところです。
